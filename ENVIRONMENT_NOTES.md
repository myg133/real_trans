# 运行环境说明

## 音频设备测试注意事项

### 问题说明

您在Codespace或其他云端开发环境中运行音频测试时遇到的"0 samples"问题是正常的，因为：

1. **云端环境缺乏音频硬件**：Codespace、Docker容器、云服务器等通常没有实际的音频设备
2. **音频驱动不可用**：错误信息显示 `ALSA lib` 相关错误，表示底层音频系统不可用
3. **权限限制**：即使有音频设备，云端环境也可能限制对音频硬件的访问

### 解决方案

#### 1. 在本地计算机上运行
- 将项目克隆到您的个人电脑上
- 确保电脑有可用的麦克风和扬声器
- 运行测试命令

#### 2. 本地测试步骤
```bash
# 1. 克隆项目到本地
git clone <repository-url>
cd real_trans

# 2. 检查音频设备
cargo run --example list_audio_devices

# 3. 进行测试
cargo run --example input_test
```

#### 3. 设备选择指南
根据您之前获取的设备列表：
- **输入设备**：`插孔麦克风 (2- Realtek(R) Audio)` 或 `Microphone Array (2- Realtek(R) Audio)`
- **输出设备**：`耳机 (2- Realtek(R) Audio)` 或 `CABLE Output (VB-Audio Virtual Cable)`

### 云端环境的局限性

以下功能在云端环境中受限或无法正常工作：
- 音频输入/输出测试
- 实时音频流处理
- 物理音频设备访问

但以下功能仍可在云端测试：
- 翻译逻辑
- 文字转语音（部分）
- 数据处理流程

### 建议的工作流程

1. **开发阶段**：在云端进行代码编写和逻辑测试
2. **音频测试阶段**：在本地计算机上测试音频功能
3. **部署阶段**：部署到有音频硬件的目标设备

### 会议集成准备

当您在本地计算机上测试成功后：
1. 在会议软件中设置音频设备
2. 使用虚拟音频设备（如VB-Cable）进行集成
3. 验证实时翻译功能

---
**注意**：`recorded_input.raw` 为空文件是预期行为，当且仅当在没有音频硬件的环境中运行时。
